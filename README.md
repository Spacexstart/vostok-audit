# Сайт Восток-Аудит — инструкция

Это сайт на **Hugo**. Простыми словами: вы пишете текст в обычном текстовом файле, Hugo превращает его в HTML-страницу. Никакой базы данных, никакого PHP, никакой админки. Опубликован на [vaudit27.ru](https://vaudit27.ru/) через GitHub Pages.

## Программа-помощник (для заказчика)

Чтобы заказчику не нужно было разбираться с markdown, командной строкой и Git — в корне проекта лежит файл **`ПОМОЩНИК.bat`** (Windows-only). Двойной клик → открывается окно с меню:

```
  ╔═══════════════════════════════════════════════════╗
  ║       Помощник сайта  vaudit27.ru                 ║
  ╚═══════════════════════════════════════════════════╝

  Что вы хотите сделать?

    1) 👁  Посмотреть сайт у себя на компьютере
    2) 📅 Добавить новый год раскрытия информации
    3) 📰 Добавить публикацию в раздел Пресса
    4) 💬 Добавить отзыв клиента
    5) 🔨 Собрать готовый сайт (проверка на ошибки)
    6) 🚀 Опубликовать изменения (отправить на сайт)

    0) Выйти
```

Каждый пункт пошагово спрашивает у пользователя что нужно (год, заголовок, имя картинки, и т.д.) и сам делает всю работу: генерирует файл из архетипа, дописывает в YAML, открывает Блокнот для финальной редактуры, делает `git commit && git push`.

**Файлы:**
- [`ПОМОЩНИК.bat`](ПОМОЩНИК.bat) — лаунчер, запускает PowerShell-скрипт.
- [`scripts/меню.ps1`](scripts/меню.ps1) — собственно меню и все функции. Здесь можно добавлять новые пункты.
- [`ПРОЧИТАЙ-МЕНЯ.txt`](ПРОЧИТАЙ-МЕНЯ.txt) — короткая инструкция для заказчика (без технических деталей).

**Что должно быть установлено** на компьютере заказчика (один раз):
1. **Hugo Extended** — https://github.com/gohugoio/hugo/releases (файл со словом `extended` для Windows). Распаковать `hugo.exe` и добавить в системный PATH.
2. **Git** — https://git-scm.com/download/win (стандартная установка).
3. **Один раз настроить Git** для авторизации в GitHub: `git config --global user.name "Имя"`, `git config --global user.email "email"`. Push настроить через GitHub CLI или сохранённый токен.

После этого заказчик пользуется только `ПОМОЩНИК.bat`, в командную строку лезть не нужно.

**Что должно быть в репо для работы автодеплоя:**
- [`.github/workflows/deploy.yml`](.github/workflows/deploy.yml) — уже создан. При каждом `git push` собирает сайт и публикует на GitHub Pages. Один раз в настройках репо: **Settings → Pages → Source: GitHub Actions**.

Без этого `git push` отправит исходники, но сайт сам не пересоберётся.

**Как добавить новый пункт меню:** открыть [`scripts/меню.ps1`](scripts/меню.ps1), скопировать любую функцию (например, `Add-Review`), переписать её под новую задачу, добавить новый пункт в `switch ($choice)` и в текст меню.

## Я только хочу поменять текст

Если у вас не установлена никакая среда разработки и вы просто хотите подправить телефон или добавить отзыв — это можно сделать **прямо в браузере на GitHub**, ничего не устанавливая.

1. Зайдите в репозиторий проекта на GitHub под вашим аккаунтом.
2. Найдите нужный файл (см. таблицу [«Что где лежит»](#что-где-лежит-короткая-шпаргалка) ниже).
3. Нажмите кнопку с **карандашиком ✏️** в правом верхнем углу — откроется редактор прямо в браузере.
4. Поправьте текст (это обычный текст, плюс **жирный** делается двумя звёздочками: `**жирный**`).
5. Прокрутите вниз → кнопка **«Commit changes»** → ввести короткое описание изменения → **«Commit changes»** ещё раз.
6. Готово. Если у проекта настроен GitHub Actions (см. [деплой](#как-задеплоить-сайт)) — сайт обновится сам через ~1 минуту. Если нет — нужно будет позвать разработчика собрать и опубликовать.

### Добавить новый отчётный год раскрытия информации (без боли)

🔥 **Это самая частая ежегодная задача.** Сделана в одну команду:

```bash
hugo new --kind disclosure content/about/disclosure/2025.md
```

Hugo создаст файл `content/about/disclosure/2025.md` со всем готовым текстом раскрытия — год **2025** уже подставлен везде (заголовок, СВКК на 01.01.2026 посчитана автоматически, «информация актуальна на 31 декабря 2025» и т.д.).

После этого надо **только три правки**:

1. **Цифры выручки внизу страницы** — там стоят прочерки `___ тыс. руб.` с пометкой `<!-- TODO -->`. Подставить реальные значения и удалить TODO.
2. **(Опционально) Новые внешние проверки качества** — если за год были проверки СРО ААС / УФК, добавить пункт в список «Внешние проверки качества...».
3. **Сохранить файл** — карточка нового года автоматически появится:
   - на странице `/about/disclosure/`
   - на странице `/about/` (блок «Дополнительная информация»)
   - внизу всех disclosure-страниц и страницы истории

**Если терминала нет под рукой** — см. [подробный раздел ниже](#добавить-новый-год-раскрытия-информации) с описанием способа «руками через копирование».

## Что нужно установить

1. **Hugo Extended** версии **0.147.6 или новее**.
   Скачать: https://github.com/gohugoio/hugo/releases (взять файл с пометкой *extended* для вашей ОС).
   После установки проверить:
   ```bash
   hugo version
   ```
   Должно вывести что-то вроде `hugo v0.147.6+extended ...`. Если нет слова `extended` — переустановите ту же версию, но `extended`.

2. **Git** (если будете коммитить и деплоить через GitHub).

Больше **ничего** не нужно. Никаких npm/yarn/python.

## Быстрый старт

```bash
# 1. Перейти в папку проекта
cd vaudit27-main

# 2. Запустить локальный dev-сервер с автоперезагрузкой
hugo server
```

Откройте в браузере: **http://localhost:1313/**

Любое изменение в файлах `content/`, `layouts/`, `assets/`, `static/` автоматически перезагружает страницу. Сборка занимает доли секунды.

Чтобы остановить сервер — `Ctrl+C` в терминале.

Чтобы собрать «продакшн»-версию (готовые HTML-файлы для деплоя):

```bash
hugo --minify
```

Результат появится в папке `public/`. Это и есть готовый сайт — его нужно загрузить на хостинг (см. [Как задеплоить сайт](#как-задеплоить-сайт)).

## Структура проекта

```
vaudit27-main/
├── hugo.toml              # ⚙️  Глобальные настройки сайта (контакты, меню, домен, Яндекс-API)
├── content/               # 📝 Все тексты страниц (markdown)
├── layouts/               # 🎨 HTML-шаблоны (как страницы выглядят)
├── data/                  # 📊 Данные для повторяющихся блоков (пресса, отзывы)
├── assets/                # 🛠  Исходные CSS и JS — обрабатываются Hugo (минификация, фингерпринт)
├── static/                # 📦 Файлы, копируемые в собранный сайт «как есть» (картинки, .docx, favicon, CNAME)
├── archetypes/            # (служебная папка Hugo, можно игнорировать)
├── public/                # ⚠️  СЮДА Hugo собирает готовый сайт. В Git не коммитится — добавлено в .gitignore
├── dist-legacy/           # 🗄  Старая собранная версия сайта (см. отдельный раздел ниже)
├── .gitignore
├── LICENSE
└── README.md              # этот файл
```

---

Дальше идёт обычный markdown-текст страницы.

### Что внутри `layouts/`

HTML-шаблоны. Здесь определяется, **как** выглядит страница. Текст приходит из `content/` и подставляется в шаблон.

- `_default/baseof.html` — общий каркас всех страниц (`<html>`, `<head>`, `<body>`, шапка, футер).
- `partials/` — переиспользуемые куски: `head.html`, `header.html`, `footer.html`, `scripts.html`.
- `index.html` — главная.
- `services/`, `press/`, `reviews/`, `about/`, `disclosure/`, `contacts/` — шаблоны соответствующих разделов.
- `404.html` — страница «не найдено».

Если правите вёрстку — это сюда.

### Что внутри `data/`

YAML-файлы с повторяющимися данными. Из них строятся блоки на страницах:

- `data/press.yaml` — список публикаций для страницы `/press/`.
- `data/reviews.yaml` — список отзывов для страницы `/reviews/`.

### Что внутри `assets/` vs `static/`

- `assets/` — Hugo **обрабатывает** эти файлы: CSS и JS минифицируются, к именам добавляется хэш-отпечаток (cache-busting). Сейчас здесь:
  - `assets/css/main.css` — все стили сайта (≈2500 строк).
  - `assets/js/main.js` — переключатель темы, мобильное меню, лайтбокс картинок.
  - `assets/js/header-scroll.js` — поведение шапки при скролле.
- `static/` — Hugo **просто копирует** в собранный сайт. Сейчас здесь:
  - `static/images/` — все картинки (логотипы, фото, сканы публикаций и отзывов).
  - `static/files/` — DOCX-документы для скачивания.
  - `static/favicon.svg`, `static/CNAME`, `static/robots.txt` — служебные файлы.

## Как что-то изменить

### Поменять телефон, email, адрес

Открыть [hugo.toml](hugo.toml), найти блок `[params.contact]`:

```toml
[params.contact]
  phone1 = "+7 (4212) 32-41-12"
  phone1tel = "+74212324112"           # для href="tel:..." — без пробелов и скобок
  phone2 = "+7 (4212) 77-34-98"
  phone2tel = "+74212773498"
  phoneAccounting = "+7 (4212) 31-28-40"
  email = "vostok-audit@bk.ru"
  address = "ул. Волочаевская, 153, оф. 2-А"
  city = "г. Хабаровск"
  postalCode = "680000"
  whatsapp = "https://wa.me/+79145443498"
  telegram = "https://t.me/+79145443498"
```

Поменять значение, сохранить файл — `hugo server` пересоберёт всё автоматически. Контакты подставятся в шапку, футер, страницу контактов, schema.org-разметку.

### Поменять API-ключ Яндекс.Карт

Сейчас в [hugo.toml](hugo.toml) стоит **заглушка** `ваш_API_ключ`. Из-за этого карта в футере и на странице контактов не отображается. Получить настоящий ключ: https://yandex.ru/dev/maps/jsapi → выпустить ключ для домена `vaudit27.ru` → вставить в:

```toml
[params.map]
  lat = 48.476551
  lng = 135.069879
  yandexApiKey = "СЮДА_ВСТАВИТЬ_КЛЮЧ"
```

### Добавить новый год раскрытия информации

Например, нужно опубликовать раскрытие за **2025 год**.

**Способ 1 (рекомендуется): одной командой**

```bash
hugo new --kind disclosure content/about/disclosure/2025.md
```

Hugo создаст файл со **всей готовой структурой** — год уже подставлен везде (заголовок, СВКК на 01.01.2026 посчитан автоматически, дата «актуальна на 31 декабря 2025»). Шаблон лежит в [archetypes/disclosure.md](archetypes/disclosure.md).

После генерации откройте новый файл и:

1. Подставьте **реальные цифры выручки** в самом низу страницы (там стоят прочерки `___ тыс. руб.` и `<!-- TODO -->`).
2. При необходимости добавьте новые **внешние проверки качества** (СРО ААС, УФК) в соответствующем списке.
3. Удалите комментарии `<!-- TODO -->`, когда всё заполнено.

Карточка нового года **автоматически появится**:
- на странице `/about/disclosure/`
- на странице `/about/` в блоке «Дополнительная информация»
- внизу всех disclosure-страниц и страницы истории

**Способ 2 (вручную, без терминала): копированием файла**

## Как задеплоить сайт

Сайт хостится на **GitHub Pages**, домен — `vaudit27.ru` (см. файл [static/CNAME](static/CNAME) — он попадёт в собранный сайт).

### Вариант 1: ручной деплой (просто, но утомительно)

```bash
# 1. Собрать
hugo --minify

# 2. Содержимое папки public/ загрузить в корень репозитория на GitHub
#    (или на любой статический хостинг)
```

### Вариант 2: автодеплой через GitHub Actions (рекомендуется)

Один раз настроить — потом просто пушить изменения в `main`, сайт пересобирается и деплоится автоматически.

Создать файл `.github/workflows/deploy.yml`:

```yaml
name: Deploy Hugo to GitHub Pages
on:
  push:
    branches: [main]
permissions:
  contents: read
  pages: write
  id-token: write
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: peaceiris/actions-hugo@v3
        with:
          hugo-version: '0.147.6'
          extended: true
      - run: hugo --minify
      - uses: actions/upload-pages-artifact@v3
        with:
          path: ./public
  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```

В настройках репозитория на GitHub: **Settings → Pages → Source: GitHub Actions**.

После этого: правите markdown → `git commit && git push` → через ~1 минуту изменения на сайте.

## Папка `dist-legacy/` — что это и можно ли удалить

Когда проект передавался, исходники Hugo были утеряны и в репозитории лежал только готовый собранный сайт. Этот сайт сохранён в `dist-legacy/` для справки — на случай, если нужно сравнить, как было раньше, или подсмотреть какой-то блок.

**После того как вы убедились, что новая Hugo-сборка всё корректно генерирует и сайт развёрнут — папку `dist-legacy/` можно безопасно удалить.** В работе сайта она не участвует.

```bash
rm -rf dist-legacy/
```

## FAQ и типичные проблемы

### `hugo: command not found`
Hugo не установлен или не добавлен в PATH. Скачать с https://github.com/gohugoio/hugo/releases (брать файл с *extended* в названии).

### Карта Яндекса не показывается
В `hugo.toml` стоит заглушка `yandexApiKey = "ваш_API_ключ"`. Получить ключ: https://yandex.ru/dev/maps/jsapi и вставить туда же.

### Хочу добавить новую страницу в верхнее меню
Создать файл `content/моя-страница.md`, добавить блок `[[menu.main]]` в `hugo.toml`. Шаблон по умолчанию (`layouts/_default/single.html`) выведет markdown в простой контейнер.

### Картинка появилась в `static/images/`, но не отображается
Проверить путь — должен быть **без** ведущего `/static/`. То есть в шаблонах/yaml пишем `images/file.webp`, а не `/static/images/file.webp`.

### Я что-то сломал — сайт не собирается
Hugo выводит понятную ошибку с указанием файла и строки. Прочитать сообщение — обычно это либо опечатка в YAML/TOML (отступы, кавычки), либо ссылка на несуществующий шаблон. Если совсем не понятно — откатиться через `git`:
```bash
git status                # посмотреть что изменилось
git restore ИМЯ_ФАЙЛА     # откатить файл
```

### Где менять цвета/стили
В [assets/css/main.css](assets/css/main.css) — основные CSS-переменные в начале файла (`--primary-color`, `--secondary-color`, `--accent-color` и т.д.). Тёмная тема — отдельный блок `[data-theme="dark"]`.

### Как сделать пункт меню активным/подсвеченным
Это уже работает автоматически — в [layouts/partials/header.html](layouts/partials/header.html) текущий раздел получает класс `active`.

### Как поменять иконки
- В шапке/футере/карточках — иконки [Font Awesome 6](https://fontawesome.com/v6/search?o=r&m=free) (классы `fas fa-...`, `fab fa-...`).
- На страницах услуг и в карточках — [Material Icons](https://fonts.google.com/icons) (имя иконки внутри `<i class="material-icons">имя</i>` или в YAML-параметре `icon`).
